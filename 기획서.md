# 할일관리→회고 앱 기획서 (웹 버전)

## 프로젝트 개요

### 서비스 정의
**"AI 기반 개인 회고 도우미"**
- 할일 관리 데이터를 분석해 맞춤형 회고 질문 생성
- AI가 응답을 바탕으로 구조화된 회고록 작성
- 정기적 회고 습관 형성 지원 (주간 회고 중심)

### 목표
- **제1목표**: 실사용자 모집
- **제2목표**: 수익화 (후순위)
- **진행방식**: 사이드 프로젝트, 빠른 MVP 수립과 검증 기반 애자일

### 역할 분담
- **개발 리더**: 백엔드 개발 (Kotlin + Spring Boot + JPA), 프론트엔드 개발 (TypeScript + React + Next.js), AI 에이전트 활용
- **기획자/마케터**: 전반적인 기획과 홍보 전략 수립, 사용자 피드백 기반 후속 기획

## 타겟 사용자 및 플랫폼

### 타겟 우선순위
**1차 타겟: 개발자**
- 스프린트/주간 회고 문화에 익숙
- 생산성 도구에 대한 니즈와 지불 의향 높음
- 기존 개발자 커뮤니티 네트워크 활용 가능
- 세부 타겟:
  - 스타트업/중소기업 개발자 (데이터 보안 민감)
  - 개인 생산성에 관심 많은 시니어 개발자
  - 회고 문화를 도입하고 싶지만 방법을 모르는 개발자

**2차 타겟: 직장인 일반**
- 자기계발, 생산성에 관심 많은 층

### 플랫폼 전략
- **1단계**: 웹 애플리케이션 (크로스 플랫폼, 즉시 접근 가능)
- **향후 확장**: macOS/Windows/Linux 데스크탑 앱
- **핵심 이유**: 
  - AI API 토큰 관리의 편의성 (서버에서 중앙 관리)
  - 빠른 개발 및 배포
  - 모든 운영체제에서 접근 가능
  - 사용자는 브라우저만 있으면 즉시 사용 가능

## 핵심 차별점

### 데이터 보안
- **선택적 AI 전송**: 민감한 데이터는 AI 분석에서 제외 가능
- **사용자 제어**: 어떤 데이터를 AI에 보낼지 직접 선택
- **투명성**: AI 전송 데이터 명확히 표시

### 편의성과 접근성
- 별도 설치 불필요 (웹 브라우저만 있으면 사용 가능)
- AI API 토큰 관리 불필요 (서버에서 중앙 관리)
- 크로스 플랫폼 (Windows, macOS, Linux 모두 지원)

## MVP 핵심 기능

1. **할일 관리** - 서버 저장
2. **주간 회고 트리거** (금요일 중심)
3. **선택적 AI 질문 생성** - 민감 데이터 제외 옵션
4. **회고록 자동 생성** - 사용자 응답을 구조화된 텍스트로 변환
5. **회고 히스토리** - 과거 회고 조회

## 기술 스택

### 백엔드
- **언어/프레임워크**: Kotlin + Spring Boot
- **ORM**: JPA
- **데이터베이스**: PostgreSQL (또는 MySQL)
- **AI API 통합**: OpenAI GPT-4 또는 Anthropic Claude

### 프론트엔드
- **언어**: TypeScript
- **프레임워크**: React + Next.js
- **스타일링**: TailwindCSS (또는 styled-components)
- **상태관리**: React Query + Context API

### 인프라
- **호스팅**: Vercel (프론트엔드), AWS/GCP (백엔드)
- **배포**: Docker + CI/CD

## 사용자 플로우

### 1. 할일 관리 플로우 (일상적 사용: 월~목)

1. **웹사이트 접속** → 오늘의 할일 리스트 확인
2. **새 할일 추가** (모달)
3. **할일 상태 업데이트** (진행중/완료/미완료)
4. **완료 시 메모** (선택사항) - 간단한 성과/이슈 기록
5. **금요일 준비** - "금요일 회고 가능" 상태 표시

### 2. 회고 플로우 (금요일 중심)

#### 회고 시작
1. **금요일 접속** → "이번 주 회고하기" 버튼 활성화
2. **데이터 선택**:
   - 이번 주 할일 목록 표시
   - "AI 분석에서 제외할 항목 선택" 체크박스 제공
   - 민감한 프로젝트명, 고객사 관련 등 제외 가능
3. **AI 질문 생성**:
   - 선택된 항목만 서버로 전송
   - AI가 개인화된 질문 3-5개 생성
   - 제외된 항목은 "기타 업무" 형태로 포함

#### 회고 작성
4. **질문별 답변 입력** (자유 텍스트)
5. **AI 회고록 생성** - 답변을 구조화된 회고록으로 변환
6. **사용자 검토 후 저장**

#### 회고 완료
7. **회고 히스토리 저장**
8. **다음 주 할일 리스트 초기화**

### AI 질문 생성 예시

**할일 예시:**
- ☑️ [AI 분석 포함] 새로운 API 개발
- ☑️ [AI 분석 포함] 코드 리뷰 진행  
- ☐ [AI 분석 제외] A회사 프로젝트 미팅
- ☑️ [AI 분석 포함] 버그 수정

**생성되는 질문:**
- "새로운 API 개발에서 가장 어려웠던 부분은?"
- "코드 리뷰 과정에서 배운 점은?"
- "버그 수정 과정에서 예상과 달랐던 점이 있나요?"
- "기타 업무들은 어떻게 진행되었나요?" (제외된 항목들)
- "다음 주에 개선하고 싶은 부분은?"
